## Overview

**Page Purpose:** Create a new venue with its single manager account, or edit an existing venue's details, credentials, or status. One form creates the venue and its login — the manager can start working immediately.

**User Situation:** A new venue owner in District 2 is waiting for their account. Tùng fills in the venue details, sets a username and password, and hits save. 30 seconds later, the manager can log in and start setting up their menu and rooms.

**Success Criteria:** Venue + manager account created in one save. Manager can log in immediately after creation. Status changes take effect instantly (inactive = manager locked out).

**Entry Points:**
- "Add Venue" from Venue List (blank form)
- Click venue row/✏️ from Venue List (pre-filled form)

**Exit Points:**
- Save → Venue List (03.1)
- Cancel → Venue List (03.1)
- Delete → Venue List (03.1), venue removed

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default (create) | Add Venue clicked | Blank form. Username field ready | Fill and save |
| Default (edit) | Existing venue clicked | Form pre-filled. Username read-only. Password fields empty | Edit and save |
| Username checking | After typing username | Spinner next to username field | Wait for availability result |
| Username available | Username is unique | Green "✅ Available" | Proceed |
| Username taken | Duplicate username | Red "❌ Already taken" | Change username |
| Validation error | Required fields empty / password mismatch / username dup | Red border + inline messages | Correct |
| Saving | After Save click | Button spinner. Form disabled | Wait |
| Success | Save successful | Toast → redirect to Venue List | Done |
| Delete confirm (1) | Delete clicked | Dialog: "Delete {venue}? All data will be permanently removed." / "Xóa {venue}? Tất cả dữ liệu sẽ bị xóa vĩnh viễn." | Confirm or cancel |
| Delete confirm (2) | Step 1 confirmed | Input: "Type venue name to confirm" / "Nhập tên địa điểm để xác nhận" | Type name + confirm |

## Technical Notes

- One manager account per venue — username is the login credential for the venue manager
- Venue creation is transactional: venue + manager account created together or not at all
- Password is hashed server-side (bcrypt/argon2) — never stored in plaintext
- Delete is irreversible. Alternative: set to Inactive to preserve data
- Super admin cannot see venue billing data or customer records — only management metadata
- Username cannot be changed after creation (prevents login confusion)
