## Overview

**Page Purpose:** Add a brand new menu item or edit an existing one. Every field that matters for billing and display must be here — name in both languages, code, price, category, notes, and optional image.

**User Situation:** Linh is adding the 5 new spring roll dishes the owner requested, and updating the price on 3 existing beer items. She needs to enter data quickly and trust it's correct when billing picks it up tonight.

**Success Criteria:** Create item in under 30 seconds. All validation catches errors before save. Duplicate codes flagged inline. Item immediately appears in billing search after save.

**Entry Points:**
- "Add Item" from Menu Item List (blank form)
- Click item row/✏️ from Menu Item List (pre-filled form)

**Exit Points:**
- Save → Menu Item List (02.1), new/edited item highlighted
- Cancel → Menu Item List (02.1)
- Delete → Menu Item List (02.1) with item removed

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default (create) | "Add Item" clicked | Empty form. Code auto-suggest ready | Fill and save |
| Default (edit) | Existing item clicked | Form pre-filled with all fields | Edit and save |
| Validation error | Required fields empty or duplicate code | Red border + inline message on failing field | Correct and retry |
| Saving | After clicking Save | Button shows spinner. Form disabled | Wait |
| Success | Save successful | Toast "Item saved" / "Đã lưu món" → redirect to list |
| Delete confirm | Delete button clicked | Dialog: "Delete {name}? This cannot be undone." / "Xóa {name}? Không thể hoàn tác." | Confirm or cancel |
