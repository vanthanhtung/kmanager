## Overview

**Page Purpose:** Give Linh a complete view of all menu items — organized by category, searchable, with prices visible at a glance — and let her quickly add, edit, or deactivate any item. Accurate menu data is what powers error-free billing.

**User Situation:** It's a quiet Tuesday afternoon. Linh is updating the menu — 5 new dishes, a few price changes, and one item to discontinue. She needs to find items fast and make changes without disrupting tonight's operations.

**Success Criteria:** Find any item in under 3 seconds (search). Add or edit an item in under 30 seconds. Changes reflected immediately in the billing search.

**Entry Points:**
- "Menu" in main nav
- From sidebar

**Exit Points:**
- Click item row → Menu Item Detail/Edit (02.2)
- Click "Add Item" → blank Menu Item Detail/Edit (02.2)
- Nav → Rooms, Dashboard, Room Status Board

## Page Sections

### Section: Top Bar

**OBJECT ID:** `menulist-topbar`

**Search Input**

**OBJECT ID:** `menulist-search`

| Property | Value |
|----------|-------|
| Component | SearchInput, full-width |
| Placeholder EN | "Search by name or code..." |
| Placeholder VI | "Tìm theo tên hoặc mã..." |
| Behavior | Live filter — type to filter table. Matches name (EN+VI) and code |

**Add Item Button**

**OBJECT ID:** `menulist-add-btn`

| Property | Value |
|----------|-------|
| Component | Primary button |
| EN | "+ Add Item" |
| VI | "+ Thêm món" |
| Behavior | → blank Menu Item Detail/Edit (02.2) |

---

### Section: Category Filter Chips

**OBJECT ID:** `menulist-categories`

| Property | Value |
|----------|-------|
| Component | ChipGroup, horizontal scroll |
| Chips EN | All / Food / Drinks / Beer / Wine / Snacks |
| Chips VI | Tất cả / Món ăn / Đồ uống / Bia / Rượu / Ăn vặt |
| Default | All selected |
| Behavior | Click chip → filter table. Active chip highlighted |

---

### Section: Items Table

**OBJECT ID:** `menulist-table`

| Property | Value |
|----------|-------|
| Component | Data table with sortable columns |

| Column EN | Column VI | Sortable | Description |
|-----------|-----------|----------|-------------|
| Code | Mã | Yes | Short item code |
| Name | Tên | Yes | Item name in current language |
| Category | Loại | Yes | Category badge |
| Price | Giá | Yes | VND formatted |
| Status | Trạng thái | Yes | 🟢 Active / ⚫ Inactive |
| Actions | — | No | ✏️ Edit button |

**Behavior:**
- Click row → edit item (02.2) pre-filled
- Click ✏️ → edit item
- Hover → row highlight
- Inactive items shown grayed out, still visible
- Pagination: 25 per page, controls at bottom

---

### Section: Bulk Actions

**OBJECT ID:** `menulist-bulk`

| Property | Value |
|----------|-------|
| Component | Toolbar (appears when items selected) |
| Behavior | ☐ Select multiple items. Bulk actions: Activate, Deactivate, Change Category |

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Menu page opened | Full table, all active items | Search, filter, click to edit |
| Empty | No items configured | Message "No menu items yet" / "Chưa có món nào" + prominent Add Item button | Add first item |
| Loading | Initial load or after save | Skeleton table rows | Wait |
| Filtered | After search or category chip | Filtered table with result count | Browse, edit |
| No results | Search returns nothing | "No items match '{query}'" / "Không tìm thấy '{query}'" | Clear search |

---

## Technical Notes

- Item names stored in both EN and VI fields in DB
- Table shows name in current UI language
- Search matches across both language fields and code
- Status toggle: active items appear in billing search. Inactive items are hidden from billing but preserved in admin
- Category list configurable per venue
