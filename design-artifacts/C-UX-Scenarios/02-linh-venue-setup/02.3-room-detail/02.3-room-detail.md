## Overview

**Page Purpose:** View and edit a specific room's configuration — number, name, type, area, hourly rate, and status. These values directly affect billing, so changes must be clear and warn when an active session could be impacted.

**User Situation:** The owner just raised the VIP room rate from 300,000đ to 350,000đ per hour. Linh needs to update it so tonight's billing uses the new rate. Room 204 is being renovated — she needs to mark it as Maintenance.

**Success Criteria:** Rate changes apply to new check-ins only (active sessions keep their rate). Room status changes reflect immediately on the board. Changes are obvious and reversible.

**Entry Points:**
- Click room + "Edit Room" from Room Status Board
- "Rooms" in main nav → list → click room

**Exit Points:**
- Save → Room Status Board (01.2)
- Cancel → return to previous page

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Edit existing room | Form pre-filled | Edit and save |
| Rate warning | Occupied room, rate changed | Yellow banner visible | Confirm or revert |
| Validation error | Number empty/duplicate or rate ≤ 0 | Red border + inline message | Correct |
| Saving | After Save click | Button spinner | Wait |
| Success | Save successful | Toast → Room Status Board | Done |

---

## Technical Notes

- Room rate changes are versioned — old sessions keep the rate they started with
- Status changes are immediate on the board (WebSocket or poll)
- Room number must be unique within the venue
- Area/Floor/Types are configurable venue-wide settings (future: manage from a Settings page)
