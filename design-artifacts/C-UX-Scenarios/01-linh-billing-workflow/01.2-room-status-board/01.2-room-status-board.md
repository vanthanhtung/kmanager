## Overview

**Page Purpose:** Give Linh a complete live picture of every room — status, running costs, customer info, and quick actions — all from one screen. This is her command center for the entire shift.

**User Situation:** After login or any navigation back to home, LinH lands here. At peak hour, she's scanning rooms, jumping between check-in and active sessions, and closing bills — all from this hub.

**Success Criteria:** Linh can identify any room's full status in under 2 seconds. Switching context (filter, sort, navigate) is instant. Quick actions work without leaving the board.

**Entry Points:**
- After login (01.1)
- From any page via "Rooms" in main nav
- After closing a bill (back to board)

**Exit Points:**
- Click available room → Room Check-in (01.3)
- Click occupied room → Active Room / Session View (01.4)
- Click "Generate Bill" on a room → Bill Close (01.5)
- Nav → Revenue Dashboard (01.6), Menu, or Rooms admin

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | After login, rooms loaded | Full grid of room cards | Filter, search, click rooms |
| Empty (no rooms) | Venue not yet configured | Message "No rooms configured" / "Chưa có phòng nào" + link to Room setup | Navigate to Room Detail |
| Loading | Initial data fetch or filter change | Skeleton cards (gray placeholder boxes) | Wait |
| Real-time update | Room status/timer/cost changes | Individual card updates without page reload. Color change + number tick | No action needed |

## Keyboard Shortcuts

| Key | Action |
|-----|--------|
| / or Ctrl+F | Focus room search |

| Esc | Clear search / reset filters |

---

## Technical Notes

- Room data polled every 5 seconds or via WebSocket for real-time updates
- Running cost calculated client-side: elapsed_seconds × hourly_rate / 3600
- Grid view = default. List view = table with sortable columns
- Room cards animate status changes (green→red on check-in, red→blue on checkout)
- All i18n keys prefixed with `board.`
