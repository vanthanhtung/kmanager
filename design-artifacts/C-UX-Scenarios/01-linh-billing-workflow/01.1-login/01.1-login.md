## Overview

**Page Purpose:** Authenticate the venue manager with their single account credentials. On success, land on the Room Status Board ready to work.

**User Situation:** Linh arrives for her evening shift at the front desk terminal. She needs to get into the system fast — every second on the login screen is a second not managing rooms.

**Success Criteria:** Login completes in under 5 seconds. Failed attempts show clear inline errors. Language preference persists across sessions.

**Entry Points:**
- Browser opens to kmanager login URL
- Redirected here after session timeout

**Exit Points:**
- Successful login → Room Status Board (01.2)

| Property | Value |

| Purpose | Present login form in a focused, centered card |

| Property | Value |
|----------|-------|
| Component | Button group (EN | VI) |
| Behavior | onClick → switch UI language, persist to localStorage |

#### 📝 Username Input

**OBJECT ID:** `login-username`

| Property | Value |
|----------|-------|

| Component | Checkbox |
| Translation Key | `login.remember` |

#### 🔴 Error Message

**OBJECT ID:** `login-error`

| Property | Value |
|----------|-------|
| Component | Inline alert banner |

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Page load | Login form centered, empty fields | Fill username/password |
| Loading | After clicking Login | Button shows spinner, fields disabled | Wait |
| Error (invalid) | Wrong credentials | Red inline message below password | Correct and retry |
| Error (locked) | Account deactivated | Red message, form still visible | Contact admin |
| Success | Valid credentials | Brief redirect flash | → Room Status Board |

---

## Technical Notes

- Session token stored in httpOnly cookie or localStorage (if "Remember me" checked)
- Rate limit: 5 failed attempts = 30-second lockout
- All text supports EN/VI via i18n keys. Language preference stored in localStorage
- Keyboard: Tab cycles through fields, Enter submits the form
