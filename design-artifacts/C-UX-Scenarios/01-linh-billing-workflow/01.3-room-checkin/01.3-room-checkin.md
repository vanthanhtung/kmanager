## Overview

**Page Purpose:** Register a customer, assign them to a room, optionally add an initial order, and start the billing session. One click, the timer starts.

**User Situation:** A customer walks up to the front desk. Linh needs to get their info, assign a room, and start the clock — fast. The longer check-in takes, the longer the customer waits.

**Success Criteria:** Check-in completes in under 60 seconds from first keystroke to "Start Session" click. Room status updates to Occupied immediately on the board behind the scenes.

**Entry Points:**
- Clicked an available room on Room Status Board (room pre-selected)
- From "Check In" quick action on the board

**Exit Points:**
- Confirm → Active Room Session (01.4) for that room
- Cancel → Room Status Board (01.2)

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Page load with pre-selected room | Form empty except room field. Search bar focused | Fill customer info |
| Validation error | Required fields empty on submit | Red border on empty fields, inline message | Fill and retry |
| Submitting | After clicking Start Session | Button shows spinner. Form disabled | Wait |
| Success | Session created | Brief flash → Active Room Session |

## Technical Notes

- Room number pre-selected from board, shown prominently but changeable
- Initial order is optional — Linh can start with zero items and add later in Active Session View
- Session timer starts server-side on POST — prevents client clock manipulation
- Rate type affects billing calculation — stored with the session
