## Overview

**Page Purpose:** Let Linh manage active room sessions — add food/drink items to the running bill, see real-time costs, and switch between rooms instantly. This is where revenue is captured or lost.

**User Situation:** Room 302 is 45 minutes into their session. They call for more drinks. Linh needs to add items to their bill in seconds, then switch to Room 305 which is about to check out. She may have 10+ rooms active simultaneously.

**Success Criteria:** Add an item in under 5 seconds (3 keystrokes or taps). Switch between rooms instantly. Running total always accurate.

**Entry Points:**
- After Room Check-in (01.3)
- Clicked occupied room on Room Status Board (01.2)
- From sidebar switching between active rooms

**Exit Points:**
- Click "Close Bill" → Bill Close / Checkout (01.5)
- Nav → Room Status Board (01.2) or Revenue Dashboard (01.6)

## Page Sections

### Section: Sidebar — Active Rooms

**OBJECT ID:** `session-sidebar`

| Property | Value |
|----------|-------|
| Purpose | List all active rooms for instant switching. Linh never leaves this page |
| Layout | Narrow left sidebar, ~60px wide |

**OBJECT ID:** `session-room-tab`

| Property | Value |
|----------|-------|
| Component | VerticalTab list |

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Active session loaded | Menu grid + running bill. Timer ticking | Search, add items, switch rooms |
| Empty bill | Session just started, no items yet | Only room charge line visible (ticking). "No items yet" message in bill area | Search and add first item |
| Room switching | Click another room tab | Brief loading flash, then that room's bill and items load | Continue working |
| Discount applied | After adding discount | Discount line appears above grand total. Grand total recalculated | Remove discount or close bill |

## Technical Notes

- Real-time clock client-side, synced with server periodically
- Item search with debounce (200ms) for fast local filtering
- Active rooms sidebar refreshes as rooms open/close
- Grand total calculation: sum(item_qty × item_price) + (elapsed_hours × room_rate) — discount
- All monetary values in VND, formatted with thousand separators
